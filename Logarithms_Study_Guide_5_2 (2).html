Section 01

What Even IS a Logarithm?

Let's start with something you already know. You're comfortable with exponentials like:

23 = 8

This says: "2 multiplied by itself 3 times gives you 8."

Now here's the big question: What if I give you the 2 and the 8, but I hide the 3? In other words, what if I ask:

"2 to the what power equals 8?"

The answer is 3. And a logarithm is just a fancy way of writing that question.

log2(8) = 3

That's it. That's all a logarithm is. It asks: "What exponent do I need?"

💡 Real-World Analogy: Think of a logarithm like asking "how many times did I double?" If you start with $1 and keep doubling, after how many doubles do you have $8? Answer: 3 doubles. That's log2(8) = 3.
🔎 The Official Definition

For a base a (where a > 0 and a ≠ 1):

loga(x) = y   means   ay = x

We read this as: "Log base a of x equals y"

Translation: "a raised to the y power gives me x."

🔑 Why Do We Need Them?

Remember from Section 5.1: if you have f(x) = 3x and you want to find the inverse, you swap x and y to get x = 3y. But then... how do you get y down from the exponent? You can't just "undo" an exponent with basic algebra.

Logarithms are the tool that gets a variable OUT of an exponent. They are the inverse of exponential functions.

Section 02

The Swoop Method — Converting Between Forms

Your instructor taught a trick called the "swoop" and it's gold. Here's how it works. Always write the log with the answer on the right side:

loga(x) = y

Now do the swoop: start at the base a, jump up to the answer y (it becomes the exponent), and land on x:

LOG FORM
loga(x) = y
⟷
EXPONENTIAL FORM
ay = x

The base stays the base. The answer (y) becomes the exponent. The number inside the log (x) is what you get.

✅ How to Remember the Swoop

In loga(x) = y, trace a curved line from a → up to y → over to x. That gives you: ay = x.

Practice drawing this swoop on paper until it becomes automatic!

Section 03

Examples: Converting Between Forms

Exponential → Log Form

Example 1

Convert:   42 = 16   into log form.

Think: The base is 4. The exponent is 2. The result is 16.

So: "4 raised to what gives me 16?" → the answer is 2.

log4(16) = 2   ✓
Example 2

Convert:   3-2 = 1/9   into log form.

Think: Base is 3. Exponent is -2. Result is 1/9.

So: "3 raised to what gives me 1/9?" → the answer is -2.

log3(1/9) = -2   ✓

Log Form → Exponential

Example 3

Convert:   log5(5) = 1   into exponential form.

Swoop: Base is 5. Answer/exponent is 1. Number inside log is 5.

51 = 5   ✓   (makes sense!)
Example 4

Convert:   log3(x) = y   into exponential form.

Swoop: Base is 3, exponent is y, result is x.

3y = x   ✓
Section 04

Evaluating Logarithms (Solving by Hand)

To evaluate a log, ask yourself the key question: "The base raised to WHAT gives me this number?"

Example 1 — log2(8)

Ask: "2 to the what equals 8?"

21 = 2 ... nope

22 = 4 ... nope

23 = 8 ... YES!

log2(8) = 3
Example 2 — log1/3(9)

Ask: "(1/3) to the what equals 9?"

This one's tricky. When you have a fraction as a base and need a whole number result, think negative exponents.

(1/3)-2 = (3/1)2 = 32 = 9   YES!

log1/3(9) = -2

Remember: A negative exponent flips the fraction, then you apply the positive exponent.

Example 3 — f(64) where f(x) = log4(x)

We need log4(64). Ask: "4 to the what equals 64?"

41 = 4

42 = 16

43 = 64   YES!

f(64) = 3
💡 Strategy for Evaluating Logs by Hand
  1. Identify the base (the little subscript number).
  2. Identify the number inside the log (the argument).
  3. Ask: "Base to the WHAT power gives me that number?"
  4. Try powers: start with 0, 1, 2, 3... and try negatives if the number is a fraction.
Section 05

Special Logs: Natural Log & Common Log

Most logarithms can't be computed on a basic TI-83/84 calculator. But there are two special types that your calculator CAN handle:

📜 Natural Log (ln)

Base = e (where e ≈ 2.718, that special constant from Section 5.1)

ln(x)   means   loge(x)

On your calculator: press the LN button.

Example: ln(10) ≈ 2.303 (just type it in!)

Key fact to memorize: ln(e) = 1 because e1 = e.

📜 Common Log (log)

Base = 10

log(x)   means   log10(x)

On your calculator: press the LOG button (it's base 10 by default).

Example: log(15) ≈ 1.176 (just type it in!)

When you see "log" written with NO base, it always means base 10.

Practice: Common Log

Find log(100) by hand.

This means log10(100). Ask: "10 to the what equals 100?"

102 = 100 → Answer: 2

Practice: Natural Log

Find ln(1/e3) by hand.

This means loge(1/e3). Ask: "e to the what equals 1/e3?"

e-3 = 1/e3 → Answer: -3

(Negative exponent flips it to the denominator.)

⚠️ Calculator Warning

Your TI-83/84 can ONLY do ln (base e) and log (base 10). For any other base (like log2, log3, log5, etc.), you MUST solve by hand or use the change of base formula (covered in Section 5.3). Don't rely on online calculators — they won't be there on test day!

Section 06

The 4 Must-Know Properties

These are rules you need to memorize. They work for any base b (where b > 0 and b ≠ 1). Each one makes total sense if you think about what it means in exponential form.

Property 1: Log of the Base Itself
logb(b) = 1
💬 "b to the WHAT gives me b? The answer is 1, because b1 = b."
log7(7) = 1    log5(5) = 1    ln(e) = 1
Property 2: Log of 1
logb(1) = 0
💬 "b to the WHAT gives me 1? The answer is 0, because b0 = 1." (Anything to the 0 power is 1!)
log7(1) = 0    log0.37(1) = 0    ln(1) = 0
Property 3: Log of a Base to a Power
logb(bx) = x
💬 "b to the WHAT gives me bx? Obviously x!" The matching b's cancel out.
log7(78) = 8    log5(53) = 3    ln(e4) = 4
Property 4: Base Raised to Its Own Log
blogb(x) = x
💬 The exponential and the log are inverses — they cancel each other out, leaving just x.
9log9(x) = x    eln(x) = x    10log(x) = x
💡 Think of Properties 3 & 4 like this: Logs and exponentials with the same base are like an "undo button" for each other. If you do one and then the other, they cancel and you're left with what you started with. Just like how putting on your shoes and taking off your shoes leaves you where you started.
Section 07

The 3 Big Rules (Product, Quotient, Power)

These three rules let you break apart (expand) or combine (condense) logarithms. They're your main tools for working with log expressions. For all of these, the base b must be positive and ≠ 1, and M, N must be positive.

✖ Product Rule
logb(M · N) = logb(M) + logb(N)

💬 "The log of a product becomes the SUM of the logs."

Multiplication inside → Addition outside.

✖ Quotient Rule
logb(M / N) = logb(M) - logb(N)

💬 "The log of a fraction becomes the DIFFERENCE of the logs."

Division inside → Subtraction outside.

✖ Power Rule
logb(Mk) = k · logb(M)

💬 "An exponent inside the log can drop down in front as a multiplier."

Exponent inside → Coefficient outside.

💡 Memory trick: Think of the log like an elevator. Multiplication goes UP (to addition). Division goes DOWN (to subtraction). Exponents JUMP to the front.
Section 08

Expanding Logarithmic Expressions

Expanding means breaking a single log into multiple separate logs using the 3 rules. Work from the outside in: quotients first, then products, then powers.

Example 1 — log5(3x)

We see a product: 3 times x.

Apply the Product Rule:

log5(3x) = log5(3) + log5(x)

Done! That's the fully expanded form.

Example 2 — log2(x3)

We see a power: x raised to the 3.

Apply the Power Rule: bring the 3 down front.

log2(x3) = 3 · log2(x)
Example 3 — ln(a / b)

We see a quotient: a divided by b.

Apply the Quotient Rule:

ln(a/b) = ln(a) - ln(b)
Example 4 (Big one!) — log3(5x2 / 2y)

This has a quotient, products, AND a power. Take it one step at a time:

Step 1: Quotient Rule — split the fraction:

log3(5x2) - log3(2y)

Step 2: Product Rule — break each piece apart:

log3(5) + log3(x2) - log3(2) - log3(y)

(The minus sign distributes to BOTH log3(2) and log3(y)!)

Step 3: Power Rule — bring the 2 down from x2:

log3(5) + 2 · log3(x) - log3(2) - log3(y)
⚠️ #1 Mistake Students Make

log(x + 3) is NOT the same as log(x) + log(3)!

The Product Rule only works when you have multiplication inside the log. Addition inside the log CANNOT be split apart. If you see a plus sign inside the log, leave it alone.

log(x + 3) ≠ log(x) + log(3)   ❌
log(x · 3) = log(x) + log(3)   ✅
⚠️ Watch Out for Exponents on Products

In log3(5x2), the exponent 2 is ONLY on x, NOT on 5. You must separate 5 and x2 with the Product Rule FIRST, then bring down the exponent. If there were parentheses like (5x)2, that would be different — but without them, the exponent only belongs to what it's directly touching.

Section 09

Condensing Logarithmic Expressions

Condensing is the opposite of expanding. You combine multiple logs into a single log. Work the rules in reverse: powers first (move coefficients up), then products/quotients.

Example 1 — log2(x) + 3 · log2(y)

Step 1: Power Rule in reverse — move the coefficient 3 up as an exponent:

log2(x) + log2(y3)

Step 2: Product Rule in reverse — addition becomes multiplication:

log2(x · y3)
Example 2 — log(a) - log(b)

Subtraction → Quotient Rule in reverse:

log(a/b)

That's it! (Both were common logs, so they have the same base.)

💡 Condensing Steps (Quick Reference)
  1. Move coefficients up. Any number in front of a log becomes an exponent on its argument. (Power Rule in reverse.)
  2. Combine addition. Logs being ADDED become a single log with multiplication inside. (Product Rule in reverse.)
  3. Combine subtraction. Logs being SUBTRACTED become a single log with division inside. (Quotient Rule in reverse.)

Important: You can only combine logs that have the SAME BASE.

Section 10

Graphing Logarithmic Functions

Since your calculator doesn't graph most log functions directly, here's the strategy: graph the exponential inverse, then flip the coordinates.

🗒️ Graphing Steps
  1. Find the inverse. The inverse of y = loga(x) is y = ax. This is the exponential you already know how to graph.
  2. Make a table of x and y values for y = ax.
  3. Swap every (x, y) pair to get (y, x). This reflects the graph across the line y = x.
  4. Plot the swapped points and sketch a smooth curve through them.
Full Example — Graph f(x) = log2(x)

Step 1: The inverse is y = 2x. Make a table:

x-3-2-10123
2x1/81/41/21248

Step 2: Swap x and y to get the log table:

x1/81/41/21248
log2(x)-3-2-10123

Step 3: Plot these points and connect with a smooth curve.

Key Features of Every Log Graph

No matter the base, log functions always have these features:

FeatureValue
Domain (allowed x-values)(0, infinity) — x must be positive
Range (possible y-values)(-infinity, infinity) — y can be anything
x-intercept(1, 0) — always passes through this point
Vertical Asymptotex = 0 (the y-axis) — graph gets close but never touches
Direction (base > 1)Increasing — goes up from left to right
Direction (0 < base < 1)Decreasing — goes down from left to right
💬 Why Does the Graph ALWAYS Pass Through (1, 0)?

Because logb(1) = 0 for any base b. (Property 2 from Section 06!) Any number raised to the 0 power equals 1, so the log of 1 is always 0. That means the point (1, 0) is always on the graph.

Common Mistakes to Avoid

❌ log(x + y) ≠ log(x) + log(y) — Addition inside CANNOT be split.

❌ log(x - y) ≠ log(x) - log(y) — Subtraction inside CANNOT be split.

❌ log(x) / log(y) ≠ log(x/y) — Dividing logs is NOT the quotient rule.

✅ log(x · y) = log(x) + log(y) — Only MULTIPLICATION splits into addition.

✅ log(x / y) = log(x) - log(y) — Only DIVISION splits into subtraction.

Graphing Checklist

To graph y = loga(x):

1. Graph the inverse y = ax first.

2. Swap all (x, y) pairs to (y, x).

3. Plot the new points and connect smoothly.

4. Draw the vertical asymptote at x = 0 (dashed line on the y-axis).

5. The curve always passes through (1, 0).
